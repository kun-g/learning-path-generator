# Supabase 设置指南

## 1. 创建 Supabase 项目

1. 访问 [Supabase](https://supabase.com)
2. 点击 "Start your project" 或 "New Project"
3. 选择你的组织（如果没有则创建一个）
4. 项目名称：`learning-path-generator`
5. 数据库密码：设置一个强密码并保存
6. 地区：选择离你最近的地区
7. 点击 "Create new project"

## 3. 获取项目配置

1. 在 Supabase 项目仪表板，点击左侧的 "Settings"
2. 点击 "API"
3. 复制以下信息：
   - **Project URL** (类似: https://xxxxxxxxxxxxx.supabase.co)
   - **anon/public key** (很长的字符串)

## 4. 更新环境变量

将获取的信息填入 `.env.local` 文件：

```env
# OpenRouter API配置 (用于deepseek等模型)
OPENAI_API_KEY=your_openrouter_api_key_here
OPENAI_BASE_URL=https://openrouter.ai/api/v1

# Supabase配置
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_anon_key_here
```

## 5. 切换到真实数据库

目前项目使用 Mock 数据。要使用真实的 Supabase 数据库，需要：

1. 确保按上述步骤设置了 Supabase 项目
2. 更新了 `.env.local` 文件
3. 重启开发服务器：`bun run dev`

## 6. 验证设置

访问 `/blog` 页面，如果能看到文章列表，说明设置成功！

## 注意事项

- 确保 Supabase 项目的 RLS 策略已正确设置
- 检查环境变量是否正确配置
- Mock 数据和真实数据库可以无缝切换
